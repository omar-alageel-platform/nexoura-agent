NEXOURA Decision · 002

Pick primary research tool for Stage 3 (Research-First)

Context: Stage 3 currently uses three research providers ad-hoc, costing both budget and worker switching overhead. We need one primary and a documented fallback before we scale Stage 3 across tenants.

Exa

Neural search-as-API, strong for vague semantic queries. Strong "find me companies like X" use cases.

  • Best semantic recall on B2B queries
  • Cleanest API ergonomics
  • Predictable pricing
  • Weaker on dynamic-JS pages
  • No built-in scraping

Firecrawl

Scrape + crawl-first tool, returns markdown-clean pages. Strongest when we already know the URL.

  • Handles JS-rendered sites
  • Clean markdown output
  • Good for known-URL workflows
  • Weak at discovery / "find me"
  • Higher per-call cost

Parallel

Multi-agent research orchestrator that fans out across sources. Designed for deep dossiers.

  • Highest depth per query
  • Returns structured citations
  • Good for one-off deep work
  • Expensive at Stage 3 volumes
  • Slower (30–90s per query)
Recommendation: Exa — Stage 3 is dominated by discovery queries where semantic recall matters; keep Firecrawl as fallback for known-URL scraping, drop Parallel until we hit a deep-dossier workflow.
